											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">本文内容</h2>
										</nav><!-- <content> --><p>你可以在<a href="struct" data-linktype="relative-path">结构类型</a>的声明中使用 <code>ref</code> 修饰符。 <code>ref struct</code> 类型的实例是在堆栈上分配的，不能转义到托管堆。 为了确保这一点，编译器将 <code>ref struct</code> 类型的使用限制如下：</p>
<ul>
<li><code>ref struct</code> 不能是数组的元素类型。</li>
<li><code>ref struct</code> 不能是类或非 <code>ref struct</code> 的字段的声明类型。</li>
<li><code>ref struct</code> 不能实现接口。</li>
<li><code>ref struct</code> 不能被装箱为 <a href="/zh-cn/dotnet/api/system.valuetype" class="no-loc" data-linktype="absolute-path">System.ValueType</a> 或 <a href="/zh-cn/dotnet/api/system.object" class="no-loc" data-linktype="absolute-path">System.Object</a>。</li>
<li><code>ref struct</code> 不能是类型参数。</li>
<li><code>ref struct</code> 变量不能在 <a href="../operators/lambda-expressions" data-linktype="relative-path">Lambda 表达式</a>或<a href="../../programming-guide/classes-and-structs/local-functions" data-linktype="relative-path">本地函数</a>中捕获。</li>
<li>在 C# 13 之前，<code>ref struct</code> 变量无法在 <code>async</code> 方法中使用。 从 C# 13 开始，<code>ref struct</code> 变量不能在 <a href="../keywords/async" data-linktype="relative-path"><code>async</code></a> 方法中和 <a href="../operators/await" data-linktype="relative-path"><code>await</code></a> 表达式用于相同的块。 但是，可以在同步方法中使用 <code>ref struct</code> 变量，例如，在返回 <a href="/zh-cn/dotnet/api/system.threading.tasks.task" class="no-loc" data-linktype="absolute-path">Task</a> 或 <a href="/zh-cn/dotnet/api/system.threading.tasks.task-1" class="no-loc" data-linktype="absolute-path">Task&lt;TResult&gt;</a> 的方法中。</li>
<li>在 C# 13 之前，<code>ref struct</code> 变量不能在<a href="../../iterators" data-linktype="relative-path">迭代器</a>中使用。 从 C# 13 开始，<code>ref struct</code> 类型和 <code>ref</code> 局部变量可以在迭代器中使用，前提是它们不在代码段中具有 <code>yield return</code> 语句。</li>
</ul>
<p>可以定义一次性的 <code>ref struct</code>。 为此，请确保 <code>ref struct</code> 符合<a href="../proposals/csharp-8.0/using#pattern-based-using" data-linktype="relative-path">一次性模式</a>。 也就是说，它有一个实例 <code>Dispose</code> 方法，该方法是可访问、无参数的并且具有 <code>void</code> 返回类型。 可以将 <a href="../statements/using" data-linktype="relative-path">using 语句或声明</a>与可释放的 <code>ref struct</code> 的实例一起使用。</p>
<p>通常，如果需要一种同时包含 <code>ref struct</code> 类型的数据成员的类型，可以定义 <code>ref struct</code> 类型：</p>
<pre><code class="lang-csharp">public ref struct CustomRef
{
    public bool IsValid;
    public Span&lt;int&gt; Inputs;
    public Span&lt;int&gt; Outputs;
}
</code></pre>
<p>若要将 <code>ref struct</code> 声明为 <code>readonly</code>，请在类型声明中组合使用 <code>readonly</code> 修饰符和 <code>ref</code> 修饰符（<code>readonly</code> 修饰符必须位于 <code>ref</code> 修饰符之前）：</p>
<pre><code class="lang-csharp">public readonly ref struct ConversionRequest
{
    public ConversionRequest(double rate, ReadOnlySpan&lt;double&gt; values)
    {
        Rate = rate;
        Values = values;
    }

    public double Rate { get; }
    public ReadOnlySpan&lt;double&gt; Values { get; }
}
</code></pre>
<p>在 .NET 中，<code>ref struct</code> 的示例分别是 <a href="/zh-cn/dotnet/api/system.span-1" class="no-loc" data-linktype="absolute-path">System.Span&lt;T&gt;</a> 和 <a href="/zh-cn/dotnet/api/system.readonlyspan-1" class="no-loc" data-linktype="absolute-path">System.ReadOnlySpan&lt;T&gt;</a>。</p>
<h2 id="ref-fields"><code>ref</code> 字段</h2>
<p>从 C# 11 开始，可以在 <code>ref struct</code> 中声明 <code>ref</code> 字段，如以下示例所示：</p>
<pre><code class="lang-csharp">public ref struct RefFieldExample
{
    private ref int number;

    public int GetNumber()
    {
        if (System.Runtime.CompilerServices.Unsafe.IsNullRef(ref number))
        {
            throw new InvalidOperationException("The number ref field is not initialized.");
        }

        return number;
    }
}
</code></pre>
<p><code>ref</code> 字段可以具有 <code>null</code> 值。 使用 <a href="/zh-cn/dotnet/api/system.runtime.compilerservices.unsafe.isnullref#system-runtime-compilerservices-unsafe-isnullref-1(-0@)" class="no-loc" data-linktype="absolute-path">Unsafe.IsNullRef&lt;T&gt;(T)</a> 方法确定 <code>ref</code> 字段是否为 <code>null</code>。</p>
<p>可通过以下方式将 <code>readonly</code> 修饰符应用于 <code>ref</code> 字段：</p>
<ul>
<li><code>readonly ref</code>：只能在构造函数或 <a href="../keywords/init" data-linktype="relative-path"><code>init</code> 访问器</a>中使用 <code>= ref</code> 运算符<a href="../operators/assignment-operator#ref-assignment" data-linktype="relative-path">通过 ref 重新赋值</a>此类字段。 可以在字段访问修饰符允许的任何时间点使用 <code>=</code> 运算符分配值。</li>
<li><code>ref readonly</code>：在任何时候，都不能使用 <code>=</code> 运算符为此类字段赋值。 但是，可以使用 <code>= ref</code> 运算符通过 ref 重新赋值字段。</li>
<li><code>readonly ref readonly</code>：只能在构造函数或 <code>init</code> 访问器中通过 ref 重新赋值此类字段。 在任何时候，都不能为字段赋值。</li>
</ul>
<p>编译器确保存储在 <code>ref</code> 字段中的引用的生存期不会超过其引用。</p>
<p><code>ref</code> 字段功能支持安全实现类型，例如 <a href="/zh-cn/dotnet/api/system.span-1" class="no-loc" data-linktype="absolute-path">System.Span&lt;T&gt;</a>：</p>
<pre><code class="lang-csharp">public readonly ref struct Span&lt;T&gt;
{
    internal readonly ref T _reference;
    private readonly int _length;

    // Omitted for brevity...
}
</code></pre>
<p><code>Span&lt;T&gt;</code> 类型存储一个引用，通过该引用访问内存中的连续元素。 通过使用引用，<code>Span&lt;T&gt;</code> 实例可以避免复制它所引用的存储。</p>
